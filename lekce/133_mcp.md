# Lekce 133: MCP — Model Context Protocol

## Co je MCP?

**MCP (Model Context Protocol)** je otevřený standard od Anthropic (2024), který definuje jednotné rozhraní pro to, jak LLM aplikace komunikují s externími zdroji dat a nástroji.

Analogie: MCP je pro AI to, co **USB** je pro hardware — jeden standard, který funguje s libovolným zařízením (serverem) i hostitelem (aplikací).

```
┌─────────────────────────────────────────────────┐
│                   MCP Host                       │
│  (Claude Desktop, IDE plugin, vlastní app…)      │
│                                                  │
│   ┌──────────────┐      ┌──────────────────┐    │
│   │  MCP Client  │◀────▶│  MCP Client      │    │
│   │  (připojení  │      │  (připojení      │    │
│   │   k serveru A│      │   k serveru B)   │    │
│   └──────┬───────┘      └────────┬─────────┘    │
└──────────┼──────────────────────┼───────────────┘
           │  JSON-RPC 2.0        │
    ┌──────▼──────┐        ┌──────▼──────┐
    │  MCP Server │        │  MCP Server │
    │   (soubory) │        │  (databáze) │
    └─────────────┘        └─────────────┘
```

---

## Architektura MCP

### Tři role

| Role | Zodpovědnost | Příklad |
|------|-------------|---------|
| **Host** | Spouští klienty, rozhoduje o přístupu | Claude Desktop, VS Code |
| **Client** | Udržuje připojení k jednomu serveru | Součást hosta |
| **Server** | Vystavuje resources, tools, prompts | Vlastní Python server |

### Transportní vrstvy

| Transport | Kdy použít |
|-----------|-----------|
| `stdio` | Lokální procesy (nejčastější) |
| `SSE` (HTTP) | Vzdálené servery, webové aplikace |
| `WebSocket` | Obousměrná real-time komunikace |

## Tři základní koncepty MCP serveru

### 1. Resources — čtení dat

Resources jsou **data**, která server vystavuje (jako soubory nebo databázové záznamy). Klient je může číst, ale ne měnit.

```python
from mcp.server import Server
from mcp.types import Resource, TextContent

server = Server("muj-server")

@server.list_resources()
async def seznam_resources() -> list[Resource]:
    return [
        Resource(
            uri="file:///data/konfigurace.json",
            name="Konfigurace aplikace",
            mimeType="application/json",
        )
    ]

@server.read_resource()
async def cti_resource(uri: str) -> str:
    if uri == "file:///data/konfigurace.json":
        return '{"verze": "1.0", "debug": false}'
    raise ValueError(f"Neznámý resource: {uri}")
```

### 2. Tools — volání funkcí

Tools jsou **akce**, které LLM může provést. Fungují podobně jako Anthropic tool use, ale jsou definovány na straně serveru.

```python
from mcp.types import Tool, TextContent
import json

@server.list_tools()
async def seznam_nastroju() -> list[Tool]:
    return [
        Tool(
            name="kalkulator",
            description="Vypočítá matematický výraz.",
            inputSchema={
                "type": "object",
                "properties": {
                    "vyraz": {"type": "string"}
                },
                "required": ["vyraz"],
            },
        )
    ]

@server.call_tool()
async def zavolej_nastroj(name: str, arguments: dict) -> list[TextContent]:
    if name == "kalkulator":
        try:
            vysledek = eval(arguments["vyraz"])
            return [TextContent(type="text", text=str(vysledek))]
        except Exception as e:
            return [TextContent(type="text", text=f"Chyba: {e}")]
    raise ValueError(f"Neznámý nástroj: {name}")
```

### 3. Prompts — šablony promptů

Prompts jsou **znovupoužitelné šablony** s parametry, které může host nabídnout uživateli.

```python
from mcp.types import Prompt, PromptMessage, GetPromptResult

@server.list_prompts()
async def seznam_promptu() -> list[Prompt]:
    return [
        Prompt(
            name="shrn_dokument",
            description="Shrne zadaný dokument.",
            arguments=[{"name": "text", "description": "Text k shrnutí", "required": True}],
        )
    ]

@server.get_prompt()
async def ziskej_prompt(name: str, arguments: dict) -> GetPromptResult:
    if name == "shrn_dokument":
        return GetPromptResult(
            messages=[
                PromptMessage(
                    role="user",
                    content=TextContent(
                        type="text",
                        text=f"Shrň tento dokument stručně:\n\n{arguments['text']}",
                    ),
                )
            ]
        )
    raise ValueError(f"Neznámý prompt: {name}")
```

## Celý MCP server — minimální příklad

```python
# server.py
import asyncio
from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp.types import Tool, TextContent
import math

server = Server("kalkulacka-server")

@server.list_tools()
async def tools() -> list[Tool]:
    return [
        Tool(
            name="vypocitej",
            description="Matematické výpočty (sqrt, sin, cos, pi…)",
            inputSchema={
                "type": "object",
                "properties": {"vyraz": {"type": "string"}},
                "required": ["vyraz"],
            },
        )
    ]

@server.call_tool()
async def call(name: str, arguments: dict) -> list[TextContent]:
    if name == "vypocitej":
        ns = {"sqrt": math.sqrt, "sin": math.sin, "cos": math.cos,
              "pi": math.pi, "e": math.e, "__builtins__": {}}
        result = eval(arguments["vyraz"], ns)
        return [TextContent(type="text", text=str(result))]
    raise ValueError(name)

async def run():
    async with stdio_server() as (r, w):
        await server.run(r, w, server.create_initialization_options())

if __name__ == "__main__":
    asyncio.run(run())
```

Spuštění:
```bash
pip install mcp
python server.py
```

## MCP klient — připojení k serveru

```python
import asyncio
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def main():
    params = StdioServerParameters(
        command="python",
        args=["server.py"],
    )
    async with stdio_client(params) as (read, write):
        async with ClientSession(read, write) as session:
            await session.initialize()

            # Zjisti dostupné nástroje
            nastroje = await session.list_tools()
            print("Dostupné nástroje:", [t.name for t in nastroje.tools])

            # Zavolej nástroj
            vysledek = await session.call_tool(
                "vypocitej",
                arguments={"vyraz": "sqrt(144)"},
            )
            print("Výsledek:", vysledek.content[0].text)

asyncio.run(main())
```

## MCP vs. přímé tool use — kdy co použít?

| Kritérium | Přímý tool use | MCP |
|-----------|---------------|-----|
| Rychlost prototypu | Rychlejší | Pomalejší nastavení |
| Znovupoužitelnost | Nízká | Vysoká — server sdílí více apps |
| Ekosystém | Vlastní | Hotové servery (GitHub, Slack, DB…) |
| Izolace | Kód v aplikaci | Separátní proces |
| Více klientů | Nutno duplikovat | Jeden server, N klientů |
| Vzdálený přístup | Složitější | Nativní přes HTTP/SSE |

**Pravidlo palce:**
- Jednoduché, aplikačně specifické nástroje → přímé tool use
- Znovupoužitelná integrace sdílená více aplikacemi → MCP server

## Shrnutí

- **MCP** je otevřený protokol pro komunikaci LLM aplikací s externími zdroji
- Architektura: **Host** (app) → **Client** (připojení) → **Server** (data/akce)
- Server vystavuje tři typy primitiv: **Resources** (data), **Tools** (akce), **Prompts** (šablony)
- Komunikace probíhá přes **JSON-RPC 2.0** po `stdio` nebo HTTP/SSE
- MCP je výhodné pro **znovupoužitelné integrace** sdílené více aplikacemi
