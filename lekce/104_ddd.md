# Lekce 104: Domain-Driven Design (DDD)

## Co je DDD?

Domain-Driven Design (Eric Evans, 2003) je přístup k návrhu softwaru, kde **doménový model** — pojmy a pravidla z byznysu — stojí v centru. Kód mluví jazykem expertu z dané domény.

Klíčové pojmy:

| Pojem | Anglicky | Popis |
|-------|---------|-------|
| Entita | Entity | Má identitu (id), mění se v čase |
| Hodnotový objekt | Value Object | Neměnný, porovnává se hodnotou |
| Agregát | Aggregate | Cluster entit s kořenem |
| Repozitář | Repository | Načítání/ukládání agregátů |
| Doménová událost | Domain Event | Něco důležitého se stalo |
| Doménová služba | Domain Service | Logika, která nepatří jedné entitě |

---

## Entity

Entita = objekt s **identitou**. Dva uživatelé mohou mít stejné jméno, ale jiné id — jsou různí.

```python
from dataclasses import dataclass, field
from datetime import datetime

@dataclass
class UzivatelId:
    hodnota: int

@dataclass
class Uzivatel:
    id: UzivatelId
    jmeno: str
    email: str

    def __eq__(self, other: object) -> bool:
        if not isinstance(other, Uzivatel):
            return NotImplemented
        return self.id == other.id      # entita = porovnání POUZE podle id

    def __hash__(self) -> int:
        return hash(self.id.hodnota)
```

## Value Objects (hodnotové objekty)

Value Object = neměnný, porovnává se **hodnotou**, nemá identitu. Použij `frozen=True`.

```python
from dataclasses import dataclass

@dataclass(frozen=True)
class Email:
    hodnota: str

    def __post_init__(self) -> None:
        if "@" not in self.hodnota:
            raise ValueError(f"Neplatný e-mail: {self.hodnota!r}")

    def domena(self) -> str:
        return self.hodnota.split("@")[1]

@dataclass(frozen=True)
class Penize:
    castka: float
    mena: str

    def __add__(self, other: "Penize") -> "Penize":
        if self.mena != other.mena:
            raise ValueError(f"Nelze sčítat {self.mena} a {other.mena}")
        return Penize(self.castka + other.castka, self.mena)

    def __str__(self) -> str:
        return f"{self.castka:.2f} {self.mena}"
```

Value Objects jsou bezpečné pro sdílení a ukládání do `set` / klíčů `dict`.

## Doménové události (Domain Events)

Událost popisuje **co se stalo** — vždy v minulém čase. Je neměnná.

```python
from dataclasses import dataclass, field
from datetime import datetime
import uuid

@dataclass(frozen=True)
class DomenovaUdalost:
    id: str = field(default_factory=lambda: str(uuid.uuid4()))
    cas: datetime = field(default_factory=datetime.now)

@dataclass(frozen=True)
class ObjednavkaVytvorena(DomenovaUdalost):
    objednavka_id: int = 0
    zakaznik_id: int = 0
    celkova_cena: float = 0.0

@dataclass(frozen=True)
class ObjednavkaZaplacena(DomenovaUdalost):
    objednavka_id: int = 0
    castka: float = 0.0
```

## Agregáty

Agregát = **cluster entit** spravovaný jedním kořenem (Aggregate Root). Zvenčí přistupuješ jen ke kořeni.

```python
from __future__ import annotations
from dataclasses import dataclass, field
from enum import Enum, auto

class StavObjednavky(Enum):
    NOVA = auto()
    ZAPLACENA = auto()
    ZRUSENA = auto()

@dataclass
class PolozkaObjednavky:
    nazev: str
    cena: Penize
    mnozstvi: int

    def celkem(self) -> Penize:
        return Penize(self.cena.castka * self.mnozstvi, self.cena.mena)

@dataclass
class Objednavka:
    """Kořen agregátu. Veškeré změny jdou přes jeho metody."""

    id: int
    zakaznik_id: int
    polozky: list[PolozkaObjednavky] = field(default_factory=list)
    stav: StavObjednavky = StavObjednavky.NOVA
    _udalosti: list[DomenovaUdalost] = field(default_factory=list, repr=False)

    def pridej_polozku(self, nazev: str, cena: Penize, mnozstvi: int) -> None:
        if self.stav != StavObjednavky.NOVA:
            raise ValueError("Nelze přidat položku do uzavřené objednávky")
        self.polozky.append(PolozkaObjednavky(nazev, cena, mnozstvi))

    def celkova_cena(self) -> Penize:
        if not self.polozky:
            raise ValueError("Objednávka nemá žádné položky")
        celkem = self.polozky[0].celkem()
        for p in self.polozky[1:]:
            celkem = celkem + p.celkem()
        return celkem

    def zaplat(self) -> ObjednavkaZaplacena:
        if self.stav != StavObjednavky.NOVA:
            raise ValueError(f"Nelze zaplatit objednávku ve stavu {self.stav.name}")
        cena = self.celkova_cena()
        self.stav = StavObjednavky.ZAPLACENA
        udalost = ObjednavkaZaplacena(
            objednavka_id=self.id,
            castka=cena.castka,
        )
        self._udalosti.append(udalost)
        return udalost

    def pop_udalosti(self) -> list[DomenovaUdalost]:
        """Vrátí a smaže nahromaděné události (pro event dispatcher)."""
        udalosti, self._udalosti = self._udalosti, []
        return udalosti
```

## Repository

Repository poskytuje iluzi kolekce v paměti — skrývá, kde jsou data fyzicky uložena.

```python
from typing import Protocol

class ObjednavkaRepository(Protocol):
    def najdi(self, id: int) -> Objednavka | None: ...
    def uloz(self, obj: Objednavka) -> None: ...
    def dalsi_id(self) -> int: ...

class InMemoryObjednavkaRepository:
    def __init__(self) -> None:
        self._store: dict[int, Objednavka] = {}
        self._seq = 1

    def najdi(self, id: int) -> Objednavka | None:
        return self._store.get(id)

    def uloz(self, obj: Objednavka) -> None:
        self._store[obj.id] = obj

    def dalsi_id(self) -> int:
        id = self._seq
        self._seq += 1
        return id
```

## Ubiquitous Language (všudypřítomný jazyk)

DDD vyžaduje, aby kód mluvil **stejným jazykem** jako doménový expert. Špatně:

```python
# ❌ technický jazyk — co je "process" nebo "flag"?
def process(flag: bool, data: dict) -> dict: ...
```

Správně:

```python
# ✅ doménový jazyk — každý termín dává smysl zákazníkovi
def potvrd_objednavku(objednavka: Objednavka) -> PotvrzeniObjednavky: ...
```

---

## Shrnutí

- **Entita** — identita přes `id`, rovnost = porovnání id
- **Value Object** — `frozen=True`, rovnost = porovnání hodnot, bez id
- **Agregát** — kořen chrání invarianty, změny jen přes jeho metody
- **Domain Event** — neměnný záznam toho, co se stalo (minulý čas)
- **Repository** — iluze kolekce, skrývá persistenci
- **Ubiquitous Language** — kód = řeč domény
